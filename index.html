Daft Punk Inspired Markov Chain Music Generator


This project is a Markov chain generator that supports both text inputs of notes and MIDI file uploads. There is a selection to toggle which input the user wishes to use, whether it be user-inputted notes or MIDI pitches. and when the text input is used there are some notes automatically filled in for the user to start with and alter as they wish.


There are three stages in this work: the input processing, the Markov chain learning stage, and sequence generation stage. When the text input is selected and the train button is clicked, the Markov will parse through and train on the notes in the text box. When the MIDI option is selected and a file is uploaded, the file will automatically process the MIDI file and train based on the file. These inputs are transformed into pitch sequences to use for training.


Once trained on either input type, the Markov chain is used as a probabilistic model to generate notes based on the previous n notes in the sequence. The user selects the order that they want the Markov chain to be (1-6), and the chain is generated based on this order and the trained chain. The lower order chains lead to more variability but less coherence and structure.


Additionally, there are other options to enhance the generated music. I was inspired by Daft Punk style music, so I added "build" and "drop" functions and drums in the background to follow EDM-style music. The build function creates a gradually changing low pass filter and removes some of the drum sounds, and the drop function goes back to the original output. The "generate" button creates a new generated chain based on the trained chain.


This project really helped me to understand Markov chains, and how the chains capture probabilities rather than structure for an entire song. The lecture on algorithmic composition in class was already very interesting, but being able to create the training and generation functions themselves was very interesting and helped me to understand how the selected order affects the generation. Implementing the MIDI upload was also very interesting, as I uploaded files of different songs I like and got to see how the MIDI processing interpreted them. Getting to hear the generation sometimes sound similar to the original files was very rewarding as well. However, there were moments when the songs I used were instrument-heavy and weren't interpreted well. I would like to work on extending this MIDI interpretation and generation to make these songs sound even more like the originals and more Daft Punk-like, especially when they had a lot of different components that got muddled in training.


A potential limitation of my work would be no musicality constraints on output. The "generate" button helps to find a chain that sounds nice, but there are no constraints on song structure to follow music theory since pitch notes are treated individually. This randomness combined with the drums and ability to input any order of notes in the text box, this sometimes leads to music generations that are less enjoyable to listen to. Implementing more structural guidelines could lead to more enjoyable music every run through.
